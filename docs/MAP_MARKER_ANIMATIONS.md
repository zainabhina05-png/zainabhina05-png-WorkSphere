# Map Marker Animations & Custom Leaflet Icons

This document explains how custom Leaflet markers are implemented and animated within the WorkSphere map components. It outlines the use of `L.divIcon()`, custom SVG-based marker rendering, marker positioning, and CSS keyframe animations.

---

## 1. Overview

WorkSphere uses custom Leaflet `divIcon` markers instead of the default Leaflet marker assets for interactive map locations.

The custom implementation provides:

- Venue markers
- Destination markers
- SVG-based marker rendering
- Smooth pulsing animations
- Custom popup positioning
- Adjustable icon sizing and anchoring

---

## 2. Custom Leaflet Icons

Custom markers are created using Leaflet's `L.divIcon()` API.

Current marker types include:

| Marker Type       | Purpose                           |
| ----------------- | --------------------------------- |
| `venueIcon`       | Displays standard venue locations |
| `destinationIcon` | Displays destination markers      |

Both marker types are generated dynamically inside the map component.

---

## 3. Marker Configuration

Each custom marker defines positioning and rendering options.

### Icon Size

The implementation specifies custom `iconSize` values to control marker dimensions.

Example:

```ts
iconSize: [32, 32],
```

Different marker types may use different dimensions depending on their visual appearance.

---

### Icon Anchor

`iconAnchor` determines which point of the marker is attached to the map coordinate.

Example:

```ts
iconAnchor: [16, 32],
```

Using anchors keeps markers accurately positioned regardless of icon size.

---

### Popup Anchor

`popupAnchor` controls where popups appear relative to the marker.

Example:

```ts
popupAnchor: [0, -32];
```

This prevents popup windows from overlapping the marker icon.

---

### Marker Shadow

Leaflet shadow assets are still referenced through the standard configuration.

Example:

```ts
shadowUrl: "/leaflet/marker-shadow.png";
```

This preserves depth while using custom icons.

---

## 4. SVG-Based Marker Rendering

Instead of using default Leaflet PNG icons, WorkSphere renders markers using HTML generated by `L.divIcon()`.

This approach allows:

- Custom SVG graphics
- Dynamic colors
- Better styling flexibility
- CSS animations
- Easier customization

Because the marker is rendered as HTML, standard CSS styling and animations can be applied directly.

---

## 5. Marker Pulse Animation

The map includes a dedicated keyframe animation for animated markers.

```css
@keyframes markerPulse {
  0% {
    transform: scale(1);
    opacity: 1;
  }
  70% {
    transform: scale(1.6);
    opacity: 0;
  }
  100% {
    transform: scale(1.6);
    opacity: 0;
  }
}
```

The animation is applied using:

```css
animation: markerPulse 2.5s cubic-bezier(...) infinite;
```

This creates a continuously pulsing effect that draws attention to important map locations.

---

## 6. Relationship with Global Animations

The project also contains reusable animation utilities such as:

- `pulseGlow`
- `gradient`
- `fadeInUp`
- `shimmer`

The map component uses its own dedicated `markerPulse` animation for marker-specific behavior.

---

## 7. Implementation Flow

The overall rendering flow is:

1. Create a custom marker with `L.divIcon()`.
2. Generate HTML containing the SVG marker.
3. Configure marker sizing and anchors.
4. Apply the `markerPulse` animation.
5. Render the marker on the Leaflet map.
6. Display popups using configured popup anchors.

---

## Best Practices

- Prefer `L.divIcon()` when creating new custom markers.
- Keep marker dimensions consistent using `iconSize`.
- Update `iconAnchor` whenever marker dimensions change.
- Adjust `popupAnchor` if popup positioning changes.
- Reuse the existing `markerPulse` animation for consistency.
- Prefer SVG-based markers over static images whenever possible.

---

## Summary

WorkSphere customizes Leaflet markers by replacing the default icons with HTML-based `L.divIcon()` markers. Marker positioning is controlled through icon sizing, anchors, popup offsets, and shadow configuration, while CSS keyframe animations provide smooth pulsing effects for important locations on the map.
